# Handoff — 2026-03-12

## Summary

Removed PTT (Push-to-Talk) mode entirely. The app now operates with a single toggle-based open mic mode using Right Control to start/stop recording. This eliminates significant complexity: command latching, forceEnd, InputMode branching, dual hotkey handling, and OperatingMode state.

## What Changed

### Removed
- `InputMode` enum (`.dictation`, `.command`)
- `SessionController.OperatingMode` enum (`.ptt`, `.open`)
- `HotkeyEvent.pttBegan`, `.pttEnded(mode:)`, `.modeToggled` → replaced with single `.toggle`
- `HotkeyBindings.pttKey`, `.commandModifier`, `.modeToggle` → replaced with single `.toggleKey`
- `HUDState.recording(mode:, partialText:)` → now `.recording(partialText:)` (no mode)
- `AudioPipeline.forceEnd()` protocol method and implementation
- `IntentResolver.resolve(_:mode:)` mode parameter → now `resolve(_:)`
- `NSEventHotkeyManager` PTT/shift state tracking
- `SessionController.operatingMode`, `pendingInputMode`, `toggleMode()`
- All PTT-specific tests across 6 test files (~40 tests removed)

### Simplified
- `NSEventHotkeyManager`: single `toggleKeyDown` bool, handles only toggle key
- `SessionController.handle()`: single `.toggle` case — start or stop
- `SessionController.runSegmentLoop()`: always auto-restarts after processing (was OPEN-only)
- `EgregoreIntentResolver.resolve()`: dictation-style isolation is the only path
- `HUDWindow`: "Listening" instead of "Listening (PTT)" / "Listening (Open)"
- `MenuBarView`: single "Toggle mic" hotkey line

### Docs updated
- `specs.md`: rewritten for toggle-only mode
- `tasks.md`: PTT references removed
- `progress.md`: PTT references removed, PTT removal noted
- `README.md`: single mode description

## Verification State

- `swift build` passes clean
- Tests have pre-existing async stream hang issues unrelated to this change
